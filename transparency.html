Transparency

Security questions are fair, especially for an EVE Online tool. This page explains what MMOBASE uses, how EVE SSO works, what is stored, what is not stored, and how you stay in control of your character access.

MMOBASE never sees, asks for, or stores your EVE Online password.

The short version

MMOBASE uses CCP's official EVE SSO and ESI API. You approve specific permissions through CCP, MMOBASE receives an access token, and the dashboard uses that token to display the data you approved.

No EVE password access

You log in through CCP, not through MMOBASE. Your EVE password never touches this site.

Dashboard data only

Character, wallet, assets, market, training and location data are used to power your dashboard views.

You can revoke access

If you remove MMOBASE from your CCP authorised applications, future ESI access for that character stops.

How EVE SSO works

MMOBASE does not log into your EVE account. You log into CCP, CCP asks you to approve specific ESI permissions, and MMOBASE receives a token that can read the approved data. That data is then turned into dashboards, trends, summaries and saved snapshots. If you revoke permission later, MMOBASE can no longer refresh that character's ESI data.

What MMOBASE uses

The aim is to use only the data needed for the product. Real dashboard values should come from approved ESI data or saved MMOBASE snapshots.

Character data

  • Character name, ID and portrait
  • Corporation and alliance display
  • Security status and attributes where relevant
  • Current ship and location when approved

Training data

  • Total skill points
  • Skill list and skill levels
  • Current skill queue
  • Clone and attribute context where useful

Finance and assets

  • Wallet balance and wallet journal
  • Wallet transactions where available
  • Assets, ships and item locations
  • Market orders and estimated item values

How the data flow works

The important part is that CCP handles the EVE login. MMOBASE receives approved access, not your password.

You choose to connect

You click the connect/login option from MMOBASE.

CCP handles login

You log in on CCP's official side and review the permission prompt.

MMOBASE receives a token

The token allows MMOBASE to request only the ESI data you approved.

The dashboard is built

MMOBASE formats that data into tabs, summaries, charts and snapshots.

What MMOBASE does not do

This is just as important as what the app does use.

Account safety

  • Does not see or store your EVE Online password
  • Does not ask you to type your EVE password into MMOBASE
  • Does not control your EVE client
  • Does not sell your character data to advertisers

In-game safety

  • Does not move your ISK
  • Does not move your items
  • Does not sell your ships, contracts or market items
  • Does not make in-game decisions for you

What MMOBASE stores

To keep the dashboard working after you connect a character, MMOBASE stores the minimum account and EVE connection data needed to refresh and display the dashboard.

Account data

MMOBASE account/session data is handled through Supabase auth so users can sign in and manage their linked characters.

EVE connection data

Linked character IDs, names and server-side EVE SSO tokens are stored so approved ESI data can refresh.

Dashboard snapshots

Daily wallet and asset snapshots are saved so 7, 14 and 30 day trend charts can build over time.

Trust commitments

MMOBASE is not open source at this stage, but transparency still matters. These are the standards the project should be held to.

Plain-language permissions

Users should understand why a scope is requested before connecting a character.

Scope cleanup before wider release

Permissions that are not required by live features should be removed or clearly separated.

Clear data sources

Dashboard values should come from approved ESI data, saved snapshots, or clear loading and empty states.

Easy revoke path

Users should be able to revoke EVE access through CCP and disconnect/delete data through MMOBASE account tools.

Open source question

Some players prefer tools to be fully open source, and that is completely understandable in EVE. MMOBASE is not open source right now because it is being built as a product that may need to generate revenue in the future.

Instead, the goal is to be transparent about permissions, data use, limitations, security decisions and roadmap changes. A private or independent code review may also be considered later as the project matures.

How to revoke access

  • Go to EVE Online account management
  • Find third-party or authorised applications
  • Remove MMOBASE access for the relevant character
  • Return to MMOBASE settings to remove local linked data when available

For Reddit and security reviewers

Security concerns are welcome. If you are reviewing MMOBASE from a Reddit post, the most useful feedback is specific: which permission concerns you, what data you believe should not be stored, or what wording is unclear. The goal is to make the project safer and more understandable before a wider release.

Current priority before wider release: reduce unused scopes, document what each tab uses, and make disconnect/delete controls obvious.
